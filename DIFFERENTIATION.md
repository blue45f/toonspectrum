# ToonSpectrum 차별화 전략 — "고르는 단계"를 소유하는 큐레이터 인덱스

## 0. 한 문장 포지셔닝

> **"플랫폼 위에 올라타, 무엇을·어디서·왜 볼지를 신뢰 가능한 데이터로 답하는 큐레이터형 인덱스."**

경쟁군은 셋이고 ToonSpectrum는 그 교집합의 빈자리에 있다.
- **플랫폼 자체 앱(네이버·카카오·리디)** = 콘텐츠 호스팅 + 가두기. ToonSpectrum는 호스팅하지 않고 **가로지른다**.
- **라프텔류 추천** = 단일 카탈로그 안 추천. ToonSpectrum는 **크로스플랫폼** 취향.
- **MAL·Letterboxd류 트래킹** = 글로벌·서양 IP. ToonSpectrum는 **한국 웹툰/웹소설 + 노블코믹스 파이프라인**에 특화.

핵심 통찰: 한국 발행 플랫폼은 모두 walled garden이고, ① 크로스플랫폼 "어디서 봐"가 전무 ② 별점이 붕괴/폐지/인플레로 신뢰 불가 ③ 원작↔웹툰↔영상 관계를 데이터로 노출 안 함 ④ 대중용 트렌드 대시보드 부재(WAS는 B2B로 이탈). 이 4개 공백이 ToonSpectrum의 전 전략을 떠받친다.

## 1. 핵심 차별점 — 현 구현 자산 vs. 진짜 해자

이미 코드에 들어와 있는 것을 정직하게 구분한다(과대평가 금지).

| 자산 | 현재 상태 | 해자 등급 |
| --- | --- | --- |
| 18개 플랫폼 통합 카탈로그(~23,775편) | 실구현. `lib/server/catalog-sources.ts` 18소스, 각 ≥100편 | **A (구조적 해자)** |
| "어디서 봐" 라우터 | 실구현(연재 위치·가격모델). 단 단가는 **모델 추정**(`lib/pricing.ts`) | **A (1순위 효용)** |
| 투명 산식 다축 랭킹(8축) | 실구현. `lib/ranking.ts` 베이즈+도달가중+신뢰계수, 산식 화면 공개 | **A (가장 방어가능)** |
| 18색 장르 스펙트럼 | 실구현. `lib/genre-color.ts` 단일 출처 | **B (브랜드 해자)** |
| 어댑테이션 그래프 | 실구현. 단 **DB 내부 관계만**, 버전 간 차이(결말/매핑화수) 없음 | **B (확장 여지 큼)** |
| 신뢰 별점/리뷰 + 스포일러·태그 | 실구현(Neon). 단 콜드스타트·테러방어 미완 | **B** |
| 취향 프로필/추천 | 실구현(`lib/recommend.ts`). 단 온보딩 테스트 없음, localStorage only | **C (강화 필요)** |
| 연재 알림 구독 | **로컬 토글만**(`subscribe-button.tsx` → store). **실제 발송 없음** | **C (반쪽)** |

**진짜 방어가능한 해자(3개로 압축):**

1. **데이터 정합성 + 정직성(Data Honesty).** 단순 통합이 아니라 *어떻게* 통합하느냐가 해자다. ① 1작품=1엔트리 정본 병합(이형 제목·중복 연재처) ② 추정값을 ≈/배지로 정직하게 분리(네이버 viewCount=0 → `statsEstimated`) ③ **산식 전면 공개**. 노벨피아·카카오의 불투명 랭킹과 정반대 포지션 → "믿을 수 있는 인덱스"라는 신뢰 자산은 카피하기 가장 어렵다.
2. **노블코믹스 그래프(원작↔웹툰↔영상).** 한국 시장 고유의 IP 파이프라인. 네이버시리즈조차 *관계를 데이터로 노출하지 않는다*. 여기를 깊게 파면 한국에서 유일.
3. **크로스플랫폼 취향 + 연재 트래킹 허브.** 각 플랫폼 추천은 자기 카탈로그 안에서만 작동. "18개 플랫폼을 가로지르는 단 하나의 워치리스트/취향 프로필"은 플랫폼이 구조적으로 못 만든다(자기 작품만 있으니까).

> 표지·UI 미감은 차별점이지만 *해자는 아니다*(모방 가능). 해자는 **데이터 신뢰 + 한국 IP 관계 그래프 + 크로스플랫폼 사용자 상태**다.

## 2. 더 밀어야 할 시그니처 모먼트

브랜드를 각인시키는 "이건 ToonSpectrum다" 순간. 친숙함 위의 독창성 원칙(예측가능한 내비 + 시그니처 모먼트).

1. **"어디서 봐" 라우터를 작품 상세의 절대 1급 시민으로.** 현재 가격은 모델 추정 → **솔직하게 라벨링**하되(예: "건별 유료 · 회차 추정 ~100원, 모델 기준"), 무료/기다무/유료 신호판을 검색·랭킹·카드 어디서나 글랜서블하게. 사용자가 가장 먼저 보는 정보 = "이거 공짜로 볼 수 있나?".
2. **스펙트럼 바를 데이터 표현의 보편 문법으로.** 평점분포·장르믹스·트렌드·가격구성 전부 같은 모티프로. 한 번 보면 "ToonSpectrum 차트"로 인식되게.
3. **인덱스 넘버럴 + 산식 펼치기(why this rank).** 랭킹 행을 누르면 "이 순위인 이유"를 산식 기여도 분해로. 투명성을 *인터랙션*으로 만든다(다른 곳엔 절대 없는 것).
4. **어댑테이션 그래프를 "버전 비교"로 진화.** 단순 노드 연결 → 원작/웹툰/영상 각 버전의 **평점·완결여부·결말 차이 여부·매핑 회차**를 한 줄에. "원작이 더 평 좋음" 같은 한 줄 신호.

## 3. 신규 기능 — 차별화를 키우는 순서

경쟁 분석 Top10과 현 코드 갭을 교차해 *저비용·고차별* 순으로.

### Tier 1 — 즉시(저비용·고차별, 기존 자산 재활용)

- **연재 알림 허브 "실제 작동화" (P0).** 지금 `subscriptions`는 토글만이고 발송이 없다. 정적 캘린더 메타(`updateDays`)를 이용해 **클라이언트 측 "오늘 업데이트" 다이제스트 + 밀린 화수 트래킹**부터. (서버 푸시는 Tier 3.) MangaUpdates/AniList "다음 화" 패턴의 한국판, 요일 연재 문화에 완벽 적합.
- **취향 테스트 온보딩 (P0).** `buildTasteProfile`/`recommendForTaste`는 이미 있다 → 가입 직후 "장르·태그·코드 5문항" 콜드스타트 입력 UI만 추가. 라프텔 검증 패턴. 추천 품질이 즉시 점프.
- **"why this rank" 산식 분해 패널 (P0).** `rawScore` 기여 항목을 화면에 분해. 투명성 해자를 *눈에 보이게*. 백엔드 변경 없이 프론트만.
- **5종 상태 리스트 + 공개 컬렉션 (P0).** `ReadState`(want/reading/done/dropped) + collections 이미 존재 → "보류(hiatus)" 추가하고 컬렉션 **공개 URL** 부여. Letterboxd식 바이럴 루프 시작점.

### Tier 2 — 단기(해자 강화)

- **가격변동/무료전환 추적 (P1, 고차별).** 카탈로그가 스냅샷+크론 아키텍처 → **스냅샷 간 diff로 "기다무 전환·완전무료 전환·연재→완결" 이벤트 타임라인** 생성. "이 작품 지금 무료 풀렸어요" 알림은 한국에 *전무*. 실단가 크롤 없이 *모델 전환*만 추적해도 강력(정직성 유지).
- **어댑테이션 버전 비교 강화 (P1).** 그래프에 버전별 평점/완결/매핑화수 표시. 노블코믹스 해자 심화.
- **통합 키워드/태그 파인더 (P1).** 리디 키워드검색 30% 사용률이 수요 증명 → 전 플랫폼 작품에 일관 태그(소재·관계·전개·분위기·**트리거워닝**) 파인더 IA. `tags` 필드 이미 존재.
- **리뷰 신뢰 엔진 (P1).** 베이즈는 랭킹엔 있으나 리뷰 표시엔 미적용 → 검증 독자 가중·분포 표시·테러 방어. "별점이 고장난 시장"의 정면 공략.

### Tier 3 — 중기(인프라 동반)

- **크로스플랫폼 서버 푸시 알림 (P2).** Web Push/이메일로 "오늘 업데이트·무료 전환·완결". 사용자 락인의 핵심(트래킹이 쌓일수록 떠나기 어려움).
- **대중용 트렌드 대시보드 확장 (P2).** `/insights` 위에 "지금 뜨는 회귀물·키워드 히트맵·작가 흥행 이력". WAS의 B2B 공백을 B2C로. 작가지망생·헤비독자·기자 유입 채널.
- **팔로우 피드 + 다이어리 (P2).** 소셜 그래프. 리뷰가 콘텐츠가 되는 루프. (커뮤니티 임계질량 필요 → 후순위.)

## 4. 우선순위 로드맵 (저비용·고차별 우선)

| 순위 | 기능 | 구현 난이도 | 예상 효과 | 의존성 |
| --- | --- | --- | --- | --- |
| 1 | "why this rank" 산식 분해 패널 | 낮음(프론트) | 높음(투명성 해자 가시화) | 없음(rawScore 재사용) |
| 2 | 취향 테스트 온보딩 | 낮음 | 높음(추천·리텐션) | recommend.ts 재사용 |
| 3 | "어디서 봐" 라우터 격상 + 정직한 가격 라벨 | 낮음 | 높음(1순위 효용) | pricing.ts |
| 4 | 연재 알림 허브(클라이언트 다이제스트) | 중간 | 높음(요일연재 적합) | updateDays·subscriptions |
| 5 | 5종 상태 + 공개 컬렉션 URL | 중간 | 중상(바이럴 시작) | store·collections |
| 6 | 가격변동/무료전환 타임라인 | 중간(스냅샷 diff) | **매우 높음(시장 전무)** | 크론 스냅샷 |
| 7 | 어댑테이션 버전 비교 | 중간 | 중상(한국 IP 해자) | adaptation-graph |
| 8 | 통합 키워드 파인더 | 중간 | 중상(검증된 수요) | tags·search |
| 9 | 리뷰 신뢰 엔진 | 중간 | 중상(별점공백) | Neon 리뷰 |
| 10 | 서버 푸시·대시보드·팔로우 | 높음(인프라) | 높음(장기 락인) | 백엔드·커뮤니티 |

**90일 묶음 제안:** Tier 1 전체(1~5) + Tier 2의 #6(무료전환 추적)을 "MVP 차별화 코어"로. 이 6개만으로 *"전 플랫폼에서 찾고 → 어디서 무료로 볼지 알려주고 → 순위 이유를 투명하게 보여주고 → 무료 풀리면 알려주는"* 단일 핵심 가치가 성립한다. 이건 어떤 단일 플랫폼도 구조적으로 줄 수 없는 조합이다.

## 5. 안티레퍼런스 가드(설계 시 반드시)

- 네온퍼플/사이버펑크 금지 → warm-ink hue 64~70 축 강제. 쿨톤은 `--color-cool`로 데이터 맥락만.
- Letterboxd 카피 금지 → 소셜은 한국 웹툰 문맥(요일연재·노블코믹스·기다무)으로 재해석, 틸/그린 컬러 차용 금지.
- 카드 무한반복 금지 → 에디토리얼 비대칭 리듬, 시그니처 모먼트로 단조로움 깨기.
- 플랫폼 가두기 UX 금지 → 모든 화면에서 "여기 말고 저기서도 봐" 신호를 1급으로.
- 정직성 우선 → 추정값은 항상 ≈/배지. 가격·조회수 과장 절대 금지(신뢰가 유일한 해자라서).
