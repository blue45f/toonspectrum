# ToonSpectrum 수익화 전략 — '고르는 단계'를 소유한 디스커버리 레이어의 머니타이제이션 설계

## 0. 출발점 — 우리가 이미 가진 것 (코드 기준)

전략을 공중에 짓지 않기 위해, 레포에 실제로 배선된 수익화 인프라부터 정리한다. 새로 만들 것과 켜기만 하면 되는 것을 구분하는 게 핵심이다.

| 인프라 | 위치 | 상태 |
| --- | --- | --- |
| 전역 수익화 토글 `monetizationEnabled` | `lib/server/app-config.ts` (기본 false, 관리자만 ON) | 완성 |
| 광고/스폰서 지면 `<AdSlot>` | `components/ad-slot.tsx` (OFF면 `return null`, ON이면 "AD·스폰서" 배지 자리표시자) | 자리표시자 (네트워크 미연동) |
| "어디서 봐" 라우터 외부링크 | `components/availability.tsx` `AvailabilityRouter` (`<a target="_blank" rel="noopener noreferrer">`) | 어필리에이트 주입 지점 (URL 가공만 하면 됨) |
| 구독 플랜 테이블 `monetization_plan` | `lib/db/schema.ts:256` (code·priceCents·perks[]·intervalDays) | 스키마 완성, 결제 미연동 |
| 크리에이터 캠페인 `creator_campaign` | `schema.ts:272` (펀딩·목표/모금액) | 스키마 완성 |
| 수익 원장 `revenue_ledger` | `schema.ts:292` (kind·status pending/approved/paid·정산 settledAt·reviewNote) | 스키마 + 정산 워크플로 완성 |
| 관리자 콘솔 (플랜/캠페인/수익/정산) | `apps/api/.../admin.controller.ts`, `src/components/admin/AdminDashboard.tsx` | 완성 |

> 결론: **자금 이동(결제·정산)만 빼면 수익화 골격은 이미 80% 있다.** 전략의 핵심은 "무엇을 새로 짓느냐"가 아니라 **"가장 운영비 효율적이고 차별점(어디서 봐 라우터·신뢰 리뷰)에 부합하는 1~2개를 먼저 켜느냐"** 다.

## 1. 제휴/어필리에이트 — "어디서 봐" 라우터의 수익화 ★ 1순위 추천

### 적합성 (★★★★★)
우리의 **1번 차별점이 곧 1번 수익원**이라는 게 결정적이다. 사용자가 "어디서 무료/기다무/유료로 보나"를 비교하고 **'보러가기'를 누르는 순간이 곧 플랫폼으로 보내는 송출 행위**다. 이건 광고가 아니라 사용자가 *원해서* 누르는 의사결정의 끝단이라 UX 침습도가 0에 가깝다. Trakt×JustWatch가 정확히 이 모델로 돈을 번다.

### 구현 난이도 (★★☆☆☆ 낮음)
- 주입 지점이 이미 한 곳에 모여 있다: `AvailabilityRouter`의 `a.url`. 여기에 `lib/affiliate.ts` 한 모듈을 끼워 `decorate(platformId, url)` 형태로 제휴 파라미터/딥링크/리다이렉트를 씌우면 된다.
- 클릭 트래킹은 `/api/go/:platformId?to=...` 형태의 경량 리다이렉트 엔드포인트(NestJS) 하나로 해결. 클릭만 `revenue_ledger`의 `kind:"affiliate_click"` 또는 별도 경량 테이블에 적재 → 정산은 제휴사 리포트와 대조.
- **'어디가 더 싼가' 비교 자체가 어필리에이트 효율을 끌어올린다**: 무료/기다무 우선 정렬(이미 구현됨)은 사용자 신뢰를 사고, 유료 소장 전환은 단가가 높다.

### 한국 시장 맥락 (중요한 제약)
- 네이버/카카오는 **공개 어필리에이트 프로그램이 사실상 없다.** 리디·문피아·노벨피아·교보·예스24(전자책)는 **쿠팡파트너스/링크프라이스/제휴마케팅 네트워크**를 통해 도서·콘텐츠 제휴가 가능한 경우가 있고, 출판사 단위 제휴는 직접 딜이 현실적이다.
- 따라서 **하이브리드 전략**이 맞다: ① 제휴 가능한 플랫폼(리디·전자책·일부 웹소설)은 어필리에이트 링크, ② 제휴 불가 플랫폼(네이버·카카오)은 일반 딥링크 + **나중에 직접 트래픽 송출 실적을 근거로 B2B 제휴 협상**. 우리가 보내는 클릭/전환 데이터 자체가 협상 카드가 된다.

### 운영비 영향 (거의 0)
정적 CDN + 리다이렉트 엔드포인트 하나. Neon 부하는 클릭 로그 append-only 한 줄. 현재 "운영비 ≈ 0" 원칙을 깨지 않는다.

### 예상 수익
초기엔 작다(트래픽 함수). 하지만 **한계비용 0 + 차별점과 정합 + 사용자 신뢰 훼손 없음**이라 가장 먼저 켤 후보. 전환 단가가 큰 유료 소장·완결본 구매로 갈수록 효율이 좋다.

### 리스크
- **약관/저작권**: 딥링크 자체는 합법 영역이나, 제휴 파라미터를 무단으로 붙이면 플랫폼 약관 위반 소지. → 제휴 계약이 있는 플랫폼만 파라미터 주입, 나머지는 순수 딥링크. `PRODUCT.md` Data Governance("운영 전 약관·제휴 가능성 검토")와 정확히 일치.
- **UX**: 리다이렉트가 느리면 신뢰 하락 → `/api/go`는 즉시 302, 트래킹은 비동기.
- **투명성**: 제휴 링크임을 고지(작은 라벨)해 우리 브랜드("정직·비기만")를 지킨다.

## 2. 비침습 광고/스폰서 슬롯 — `AdSlot` 켜기

### 적합성 (★★★☆☆)
인프라가 이미 있다(`AdSlot` + `monetizationEnabled`). 다만 우리 브랜드("큐레이터형 사서", 안티레퍼런스에 장식 배너 거부감)와 일반 디스플레이 광고(Google AdSense)는 **결이 안 맞는다.** 추천은 **네트워크 디스플레이 광고가 아니라 '큐레이션형 스폰서 작품'** 이다.

### 두 갈래
1. **스폰서드 작품 슬롯 (추천)**: 출판사/플랫폼이 신작 프로모션 비용을 내고 홈 레일·검색 결과 상단·탐색에 **명확히 "스폰서" 배지가 붙은 작품 카드**로 노출. `AdSlot`의 자리표시자를 실제 작품 카드 + "AD·스폰서" 배지로 교체. 우리 디자인 언어(타이포그래픽 커버) 안에서 노출되므로 침습도 낮음. **`creator_campaign`/관리자 콘솔로 이미 캠페인 관리가 가능** → 결제만 붙이면 됨.
2. **일반 디스플레이 광고 (비추천 또는 최후)**: AdSense는 구현 쉽지만 다크 에디토리얼 무대를 깨고, "장식용 글래스/배너 거부" 브랜드 원칙과 충돌. 운영비는 0이지만 차별점을 깎는다.

### 구현 난이도 (★★☆☆☆ 낮음~중간)
스폰서드 슬롯은 `AdSlot` → 스폰서 작품 카드 렌더로 교체 + 캠페인 노출 카운트. 광고주 셀프서브가 없으면 초기엔 관리자 수동 입력으로 충분.

### 운영비 영향 (거의 0)
정적 빌드에 스폰서 캠페인 데이터를 합치거나 클라이언트에서 가벼운 fetch.

### 예상 수익 / 리스크
- 트래픽 임계점(월 수십만 UV) 전엔 광고 단가가 낮아 수익 미미. **트래픽이 충분해진 뒤** 켜는 게 맞다.
- 리스크: 랭킹/검색 결과에 스폰서가 섞이면 "투명 산식 랭킹"(핵심 신뢰 자산)을 훼손. → **스폰서는 랭킹 순위에 절대 끼우지 않고**, 별도 슬롯 + 배지로 격리. 이게 깨지면 제품 정체성이 무너진다.

## 3. 프리미엄 구독 (ToonSpectrum+) — `monetization_plan` 활용

### 적합성 (★★★★☆)
`monetization_plan` 테이블(perks[], priceCents)이 이미 구독을 전제로 설계돼 있다. 우리 데이터 자산(다축 트렌드·취향 프로필·어댑테이션 그래프·알림)은 **헤비 독자에게 유료화 여지가 명확**하다. AniList/Letterboxd Pro 모델.

### 프리미엄 기능 후보 (운영비 0에 가까운 순)
- **광고 제거**: `monetizationEnabled` ON 환경에서 구독자에게 `AdSlot`/스폰서 숨김. 구현 거의 0.
- **고급 트렌드 대시보드**: `/insights`의 심화 버전(장르 히트맵·작가 흥행 이력·급상승 추이). 데이터는 이미 사전계산(`ranking/insights JSON`) → 정적 산출물 확장만.
- **다음 화 알림 / 연재 추적 허브**: MangaUpdates식 "밀린 화수·다음 화 알림". 알림(이메일/푸시)은 약간의 운영비 발생 → 구독 가치와 맞물려 유료화에 적합.
- **무제한 컬렉션·고급 추천·테이스트 프로필 심화**.

### 구현 난이도 (★★★★☆ 높음 — 결제가 병목)
스키마/플랜/원장/관리자 콘솔은 있으나 **실결제(토스페이먼츠/포트원 등) 연동이 없다.** 한국 정기결제는 PG 계약·심사가 필요해 시간이 든다. 결제 전엔 "유료 기능을 만들어도 돈을 못 받는" 상태.

### 운영비 / 예상 / 리스크
- 운영비: 기능 자체는 정적+사전계산이라 낮음. 알림만 약간.
- 예상 수익: 구독은 **트래픽 + 충성 사용자층 형성 후**라야 ARPU가 나온다. 초기 콜드스타트(리뷰·사용자 부족)에선 전환율이 낮다.
- 리스크: 무료로 충분히 좋아야 사용자가 남는데, 프리미엄 게이팅이 과하면 디스커버리 제품의 성장(네트워크 효과)을 막는다. → **무료 핵심 유지 + 프리미엄은 헤비 유저용 심화/편의**로 한정.

## 4. 데이터/인사이트 B2B — 출판사·스튜디오 트렌드 리포트

### 적합성 (★★★★★ 차별점·마진 최상, 단 선결조건 있음)
이게 **장기적으로 가장 마진이 크고 차별점에 가장 부합**한다. WAS→COCODA가 B2B로 떠난 자리가 비어 있고, 우리는 **18개 플랫폼을 가로지르는 통합 데이터**(플랫폼 무관 통합 지수·장르 트렌드·키워드 히트맵·어댑테이션 파이프라인)를 이미 사전계산한다. 출판사/스튜디오/에이전시/투자사가 돈을 낼 대상.

### 제품화
- 월간/분기 **트렌드 리포트(PDF/대시보드)**: "지금 뜨는 회귀물·키워드·장르 이동·플랫폼 점유 변화".
- **B2B 대시보드 시트**: 작가별 흥행 이력, 신작 성과 추적, 경쟁작 벤치마크.
- **API/데이터 라이선스**: 정제된 통합 메타데이터.

### 구현 난이도 (★★★☆☆ 제품은 중간, 영업이 진짜 난이도)
데이터 파이프라인·사전계산은 있다 → 리포트/대시보드 가공은 중간 난이도. **진짜 어려운 건 데이터 신뢰성과 영업이다.** 핵심 보조지표가 추정값(≈)이라 B2B가 돈을 내려면 **실측 커버리지·정확도 입증**이 선결.

### 운영비 / 예상 / 리스크
- 운영비: 정적 데이터 재가공이라 낮음. 영업·CS 인건비가 실비용.
- 예상 수익: **객단가 최고**(건당/월 구독 단위가 큼). 소수 계약으로도 의미 있는 매출.
- 리스크: ① **데이터 정확도/저작권** — 추정값을 실측처럼 팔면 신뢰·법적 문제. ② 수집 약관 — B2B 상업화는 데이터 출처 약관을 더 엄격히 본다. ③ 영업 리드타임이 길다. → **트래픽/데이터 성숙 후 2~3순위로 추진**.

## 5. 어댑테이션/판권 디스커버리

### 적합성 (★★★☆☆ 독특하지만 니치)
어댑테이션 그래프(원작↔웹툰↔영상화)는 강한 차별점이고, **판권/IP 거래 디스커버리**는 스튜디오·제작사에게 가치가 있다. "영상화 안 된 고평점 완결 웹소설" 같은 IP 발굴 리스트는 4번 B2B의 한 갈래로 묶는 게 자연스럽다.

### 구현/운영/리스크
- 구현: 어댑테이션 그래프 데이터가 이미 있어 IP 발굴 뷰는 가공 가능. 단 **판권 거래는 우리가 중개·자금이동까지 하면 법적·계약 복잡도가 급증**(에스크로·계약·분쟁). → 우리는 **'발굴·소개'까지만** 하고 실제 거래는 외부로 보내는 게 운영비·리스크 측면에서 맞다.
- 예상 수익: 성사 시 단가는 크나 빈도 낮고 영업 의존. **4번 B2B에 흡수**해 별도 라인으로 키우지 않는 걸 권장.

## 7. 추천 — 가장 운영비 효율적이고 차별점에 부합하는 1~2개

### 즉시(Phase 1): **① 어필리에이트 "보러가기" 라우터** ★ 최우선
**이유**: 우리의 1번 차별점(어디서 봐)이 곧 수익원이고, 주입 지점이 `AvailabilityRouter` 한 곳에 모여 있으며, 한계비용 0에 UX 침습도 0이다. "운영비 ≈ 0 + 정적 CDN" 제약을 깨지 않고, 사용자가 *원해서* 누르는 행동의 끝단이라 브랜드 신뢰("정직·비기만")도 보존된다.

**최소 착수**: `lib/affiliate.ts`(제휴 가능 플랫폼만 파라미터 주입, 나머지는 순수 딥링크) + `/api/go/:platformId` 경량 302 리다이렉트 + append-only 클릭 로그. 제휴 불가 플랫폼(네이버·카카오)은 **클릭/전환 실적을 쌓아 추후 직접 B2B 제휴 협상의 근거**로 사용.

### 병행 추진(Phase 2): **④ 데이터/인사이트 B2B** ★ 장기 본진
**이유**: 차별점(18개 플랫폼 통합 데이터·투명 산식·어댑테이션 그래프)에 가장 부합하고 **마진과 객단가가 가장 크며**, COCODA가 떠난 시장 공백을 정조준한다. 데이터 파이프라인이 이미 사전계산돼 있어 제품화 난이도가 낮다. 단 **데이터 정확도 입증과 영업 리드타임**이 선결이라, 어필리에이트로 트래픽·데이터를 성숙시키며 병행 준비하는 게 순서다. 5번(판권 디스커버리)은 이 라인의 한 기능으로 흡수한다.

### 일부러 뒤로 미루는 것
- **스폰서 슬롯(②)·프리미엄(③)**: 둘 다 **트래픽 임계점·충성 사용자층이 생긴 뒤**라야 수익이 나고, 특히 프리미엄은 실결제(PG 계약·심사) 병목이 크다. 인프라(`AdSlot`·`monetization_plan`)는 이미 있으니 **트래픽이 차오르면 토글만 켜는** 옵션으로 보유.

### 한 줄 원칙
**무료 핵심(검색·랭킹·리뷰·어디서봐)은 끝까지 무료·투명·비침습으로 지키고, 그 신뢰 위에서 ①사용자가 원해 누르는 송출(어필리에이트)과 ④그 신뢰가 만든 데이터(B2B)로 번다.** 이 둘만이 "운영비 ≈ 0 + 차별점 정합 + 브랜드 신뢰 보존"을 동시에 만족한다.
